Section 01

什么是 Power Apps Code App

Code App 是 Power Apps 于 2025 年推出并已 GA 的新应用类型。它允许开发者用标准 Web 技术 （React / Vue / 纯 JS）构建 SPA，然后部署到 Power Platform 环境中运行，享受平台的认证、 连接器、DLP、共享管控等托管能力。

核心价值

开发自由 代码优先

用 VS Code + 任意前端框架，完全控制 UI 与逻辑

平台托管 Entra 认证 + 1500+ 连接器

自动处理 SSO，直接在 JS 中调用 Dataverse / SQL / SharePoint 等

企业管控 DLP + Conditional Access

应用自动遵守组织的数据防泄漏策略、条件访问、共享限制

部署简单 npx power-apps push

一条命令将 build 产物推送到 Power Platform 环境

技术架构

运行时分三层：

  1. Your Code — 你的 React 应用（打包后的 SPA 静态资源）
  2. Power Apps Client Library — @microsoft/power-apps npm 包，暴露 initialize()、data API、connector API
  3. Power Apps Host — 管理用户认证、app 加载、错误兜底
官方资源 GitHub：microsoft/PowerAppsCodeApps
文档：aka.ms/pacodeapps
npm：@microsoft/power-apps

Section 03

差异详细分析

3.1 已对齐部分（无需改动）

React 18 Vite 6 TypeScript Tailwind CSS TanStack Query React Router Radix UI Lucide Icons Recharts Zod react-hook-form power.config.json

这些库在 Code App 中完全支持，无需更换。约占项目代码量 70%。

3.2 需替换的模块

① SDK Stub → 正式 SDK

现状：app-gen-sdk/power-apps-stub.ts 提供 initialize() 和 getContext() 的空实现，外部通过 Vite alias 映射到 @microsoft/power-apps/app。

目标：安装 @microsoft/power-apps npm 包，删除 stub 文件和 alias 配置。

影响范围：app.tsx 的 initialize() 调用、use-user.ts 的 getContext() 调用。变更量约 2-3 个文件。

② Data Shim → SDK Data Layer

现状：app-gen-sdk/data/index.ts 是一个完整的 in-memory 数据客户端，实现了 IDataClient 接口（createRecordAsync / updateRecordAsync / deleteRecordAsync / retrieveRecordAsync / retrieveMultipleRecordsAsync），数据存 localStorage。

目标：使用 pac code add-connector 或 npx power-apps init 生成的 SDK data client 替代，真实连接 Dataverse。

影响范围：所有 src/generated/services/*.ts（14 个文件）中的 getClient() 调用。接口签名高度相似，主要是 import 路径变更。

利好 自建 shim 的 API 签名（createRecordAsync / retrieveMultipleRecordsAsync 等）与官方 SDK 的 data client 接口几乎一致， 迁移时不需要改动 service 层的业务逻辑，只需替换 import 来源。
③ PWA → Power Apps 托管

现状：使用 vite-plugin-pwa 生成 Service Worker、manifest.json，支持离线和安装到主屏幕。

目标：移除 PWA 插件和相关配置。Code App 不支持 PWA，也不需要 — 应用由 Power Apps Host 加载。

影响范围：vite.config.ts 中的 VitePWA 插件配置、dev-dist/ 目录、public/ 中的 PWA 图标。

注意 Code App 目前不支持 Power Apps 移动端 App（仅浏览器访问）。若团队有强离线需求，需保留 PWA 分支作为 fallback。
④ Vite Plugin 添加

现状：Vite 配置中无 Power Apps 插件。

目标：安装 @microsoft/power-apps-vite，在 vite.config.ts 中添加 powerApps() 插件。

作用：该插件在开发时自动注入 Power Apps Host 运行时、处理 local play 的 iframe 通信、管理 connector 的认证代理。

3.3 可选调整（不阻塞迁移）

  • Jotai → Zustand：Code App starter 模板用 Zustand，但 Jotai 完全兼容，无需替换
  • motion (Framer Motion)：可继续使用，不影响 Code App 运行
  • next-themes：当前用于暗色模式切换，Code App 中可保留或改为手动 class 切换
  • Copilot Studio Direct Line 集成：可在 Code App 中继续使用，但需确认网络策略不被 DLP 拦截
Section 04

迁移步骤（分阶段）

整体策略：在同一 repo 的同一目录下原地迁移，保持 git 历史连续。分 4 个阶段，每阶段可独立验证。

Phase 1：基础设施切换

1
安装官方 SDK pnpm add @microsoft/power-apps @microsoft/power-apps-vite
2
更新 vite.config.ts 添加 powerApps() 插件，移除 VitePWA() 和 Vite alias（@microsoft/power-apps/app）
3
删除 stub 文件 移除 app-gen-sdk/power-apps-stub.ts，所有 import 改为真实 SDK 路径
4
初始化 Code App npx power-apps init --display-name "Sales Copilot Mobile" --environment-id <ENV_ID>
这会更新 power.config.json（已有的会被更新/覆盖，注意备份）
5
移除 PWA 相关资产 删除 dev-dist/、PWA icons、package.json 中的 vite-plugin-pwa

Phase 2：数据层迁移

6
添加 Dataverse 连接器 使用 pac code add-datasource 或 npx power-apps find-dataverse-api 命令添加 Dataverse 数据源
7
替换 data client import 将所有 src/generated/services/*.ts 中的 import { getClient } from '../../../app-gen-sdk/data' 替换为 SDK 提供的 data client
8
适配 IDataClient 接口差异 自建 shim 使用 IOperationResult<T> 返回值，检查 SDK 的返回值格式，可能需要在 service 层做适配
9
保留 shim 作为 fallback data/index.ts 可保留，通过环境变量切换：import.meta.env.VITE_DATA_MODE === 'local' 时使用 shim，否则使用 SDK

Phase 3：认证与上下文

10
接入 Host 认证 app.tsx 中 initialize() 已正确调用。用 SDK 的 getContext() 获取真实用户信息，替代 use-user.ts 中的 localStorage 模拟
11
条件 fallback 保留非 Power Apps 环境下的 fallback 逻辑（try/catch getContext() 失败时用本地用户），便于 npm run dev 独立开发

Phase 4：部署与验证

12
Local Play 测试 npm run dev 启动后，打开 SDK 提供的 Local Play URL，在浏览器中以 Power Apps Host iframe 模式验证
13
构建并推送 npm run build && npx power-apps push
14
Power Apps 门户验证 在 make.powerapps.com 查看应用、配置共享、验证 DLP 策略合规

Section 06

认证与安全

当前方案

无认证 / localStorage 模拟

  • use-user.ts 从 localStorage 读取用户名
  • getContext() 抛异常被 catch 后用 fallback
  • 无 token / 无 session 管理
  • 适合本地开发和演示
迁移后

Power Apps Host Entra SSO

  • Host 自动处理 Microsoft Entra ID 登录
  • SDK getContext() 返回真实用户信息
  • 连接器请求自动附加用户 token
  • 支持 Conditional Access / MFA

代码变更

// use-user.ts — 现有 fallback 模式保持不变
import { getContext } from '@microsoft/power-apps/app';

async function loadUser() {
  try {
    const ctx = await getContext();
    return {
      displayName: ctx.user.fullName,
      email: ctx.user.userPrincipalName,
      objectId: ctx.user.objectId,
    };
  } catch {
    // Standalone mode fallback
    return { displayName: 'Demo User', email: '', objectId: '' };
  }
}
Section 07

Copilot Studio 集成迁移

当前项目通过 Direct Line 协议与 Copilot Studio 通信（见 copilot-service.ts）。

迁移影响评估

组件 影响 处理
Direct Line Token Endpoint 需确认 DLP 兼容 确保 Copilot Studio endpoint 不在 DLP 阻止列表中
WebSocket 连接 可正常工作 Code App 不限制 WebSocket
copilot-agent.ts（Agent 框架） 无需变更 纯前端逻辑，不依赖 SDK
function-registry.ts 无需变更 纯前端逻辑
建议 Copilot Studio 集成可作为 Phase 3 的一部分，在基础迁移完成后单独验证。 主要风险点是组织的 DLP 策略可能拦截对外部 API（Direct Line）的请求。
Section 08

已知限制与注意事项

限制 不支持移动 App

Code App 目前只能在浏览器中运行，不支持 Power Apps 移动客户端和 Windows App

限制 不支持 Git 集成

Power Platform Git Integration 尚未支持 Code App

限制 不支持 Power BI 集成

PowerBIIntegration 函数不可用，但可嵌入 Power BI Report 中

限制 Premium License

终端用户需要 Power Apps Premium 许可证才能运行 Code App

限制 SAS IP 限制

不支持 Storage Shared Access Signature IP 限制

注意 敏感数据

发布后代码托管在公开 endpoint，不要在代码中硬编码敏感信息

关键限制 移动端：当前项目是面向外勤销售人员的移动端应用（PWA + portrait mode + touch-first UI）。 Code App 不支持 Power Apps 移动客户端，用户只能通过手机浏览器访问。 这是迁移的最大功能退化点，需与业务方确认可接受性。

Section 10

风险与回退策略

风险 概率 影响 应对
SDK data client 接口与 shim 不完全一致 中 中 在 service 层加适配器；保留 shim 作为 local dev fallback
DLP 拦截 Copilot Studio Direct Line 请求 中 高 提前与 Power Platform admin 确认 DLP 白名单
移动端体验退化（无原生 App shell） 高 中 保持响应式设计，mobile browser 仍然好用；或维护 PWA 分支
Power Apps Premium 许可证成本 确定 中 需业务方 approve 许可成本（约 $20/user/month）
第三方库在 Host iframe 中有兼容问题 低 中 Local Play 阶段提前发现并修复

回退策略

Dual-Mode 架构 迁移不需要 "一刀切"。通过环境变量 VITE_PLATFORM=powerapps|standalone 控制， 可以让同一份代码在 Code App 模式和独立 SPA 模式之间切换：
  • standalone：使用 data shim + localStorage + 无认证（当前模式）
  • powerapps：使用 SDK data client + Entra auth + Host 运行时
这样即使迁移遇到阻塞，也可以随时切回独立模式。
